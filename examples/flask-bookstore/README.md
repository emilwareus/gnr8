# Flask bookstore example

An end-to-end gnr8 lifecycle for a **Flask** service: `Flask` source → OpenAPI 3.1 + a dependency-free
Python SDK, driven entirely by the `.gnr8/` Rust `Pipeline` crate (config is code — there is no
`config.toml`).

This example demonstrates gnr8's **honest Flask typed-envelope** (the second-class Python frontend):

- **Typed** handlers + typed DTOs (`OrderInput`/`OrderConfirmation`) and the `<int:order_id>` converter
  become facts → routes, params, request/response schemas in the generated OpenAPI + SDK.
- **Genuinely untyped** surfaces — a raw `request.json` body, an unannotated `request.args.get(...)`
  query read — produce **diagnostics**, never guessed facts (rule 3). They are deliberately *absent*
  from the generated output: gnr8 documents what it can prove, not what it can't.
- The Blueprint `url_prefix` (`/orders`) is a runtime value applied as the OpenAPI base path — never
  folded into the code-derived operation paths (rule 1).

The app is parsed **statically** (pyextract reads the Python `ast`; it never imports or runs the app), so
no `pip install` is needed.

## Run

```sh
cd examples/flask-bookstore
gnr8 generate      # writes generated/openapi.yaml + generated/sdk/
gnr8 check         # dry-run: confirms the committed output is byte-identical (drift gate)
```

`generated/` is committed real `gnr8 generate` output and is regenerated byte-identically by
`make examples-check`.

## Files

- `app/` — the static Flask source (copied from `fixtures/flask-bookstore`).
- `.gnr8/src/main.rs` — the generation pipeline (the config).
- `generated/` — committed OpenAPI 3.1 document + the dependency-free Python SDK.
