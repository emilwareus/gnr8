<objective>
Promote the stringly-typed `SchemaType { kind: String, ... }` (and the `Schema.kind` / `SchemaFact.kind`
"object"|"enum" discriminator) to the closed, type-system-neutral `Type` enum from
`docs/extensibility.md` §2a, and add `nullable` as a distinct axis alongside the existing `optional`
axis — threaded in lockstep across the three contract surfaces: `analyze/facts.rs` (serde DTO) ↔
`goextract/internal/facts/facts.go` (Go json tags) ↔ `graph/mod.rs` (the IR). Neutralize every Go-ism
in field names and doc comments.

Purpose: this is the narrow waist. Every later language sidecar (Phases 2–5) emits this one neutral
facts contract and lowers through this one IR. The closed enum makes "did I handle the new case?" a
compile error rather than a runtime wrong-output bug.

Output: a neutral `Type` enum + `optional`/`nullable` field axes present and round-tripping in
`facts.rs` and `graph/mod.rs`, with the Go DTO field names matching byte-for-byte. This plan does NOT
touch the consumers (`lower/`, `gosdk/`) — that is Plan 02, which the new enum will force to update via
compile errors.
</objective>
