<objective>
Build the Rust host seam that makes the existing pipeline language-aware so a Python target
flows through the SAME facts → ApiGraph → lowering path the Go sidecar uses. This is the
load-bearing gap RESEARCH identifies: today `build_graph`/`collect` call `run_goextract`
unconditionally, so a Python fixture produces garbage. This plan adds the `python3` subprocess
driver, a single deterministic language dispatch, the typed toolchain-missing error, and the
`FastApi`/`Flask` `Source` built-ins — WITHOUT touching the Python sidecar (Plan 02) or any
snapshot.

Purpose: every downstream wave (FastAPI green, Flask green) requires `build_graph(FIXTURE_DIR)`
to actually run `pyextract` against a Python dir. This seam is that switch.
Output: `PythonToolchainMissing` error, `run_pyextract`/`pyextract_dir`, language dispatch in
`build_graph` + `collect`, and `FastApi`/`Flask` Sources — all unit-tested against the Go analog.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-02-py | Denial of Service | python3 absent / unspawnable | mitigate | spawn failure maps to typed `CoreError::PythonToolchainMissing`, never `unwrap`/`expect`/`panic` (RUST-04) |
