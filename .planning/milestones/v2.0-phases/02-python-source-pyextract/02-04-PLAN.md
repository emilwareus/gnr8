Route shape: `@bp.route("/", methods=["GET"])` parses as a Call with func = Attribute(value=Name("bp"),
attr="route"), args = [Constant("/")], keywords = [keyword("methods", List([Constant("GET")]))]. One
RouteFact PER method in the methods list. `Blueprint("orders", __name__, url_prefix="/orders")` is an
Assign; the url_prefix is recorded SEPARATELY and NEVER folded into the path (rule 1; snapshot base_path `/`).
Path param converter: `"/<int:order_id>"` → strip the converter, brace the name → `/{order_id}`, and the
`int` converter drives the param schema int64.

Status (Q1 resolution, code-derived from the HTTP method): a TYPED handler (one whose return annotation
resolves to a DTO) emits a single response with status = 201 if method == POST else 200, body = the
return-annotation DTO ref. An UNTYPED handler (no resolvable return annotation) emits `responses: []`
and a diagnostic. NEVER read the docstring.

Typed body: a parameter / local whose annotation resolves to a DTO (e.g. `order: OrderInput = OrderInput(**request.json)`)
→ request_body `{"ref_id": id}`. An untyped `request.json` read with no DTO → diagnostic + no request_body.
A `request.args.get(...)` WITH a local annotation (`status: str = request.args.get(...)`) → a typed query
param fact; WITHOUT annotation (`q = request.args.get("q")`) → diagnostic + no param.

The committed snapshot (snapshot_flask_graph__flask_graph.snap) is the BYTE TARGET. Operations:
list_orders GET / (status param query, 200 → OrderConfirmation), create_order POST / (body OrderInput,
201 → OrderConfirmation), create_order_raw POST /raw (no body, responses []), get_order GET /{order_id}
(order_id path int64, 200 → OrderConfirmation). Schemas: Availability enum [in_stock,out_of_stock],
OrderConfirmation, OrderInput (four axes + union discount), Price. THREE diagnostics (sorted by file,line,message):
- line 42: "untyped query param 'q' on GET /: read via request.args.get with no annotation; param type/required-ness under-specified, type inferred as string only"
- line 69: "untyped response on POST /raw: handler has no return annotation; response shape under-specified, no schema inferred"
- line 78: "untyped request body on POST /raw: read via request.json with no typed DTO; body shape under-specified, no schema inferred"

NOTE: in the as-built routes.py the `q = request.args.get("q")` read is line 38, `def create_order_raw`
is line 66, and `payload = request.json` is line 74 — these MUST be reconciled to 42/69/78 (Task 2).
