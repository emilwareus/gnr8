The `GoSdk` Target to CLONE (`crates/gnr8-core/src/sdk/builtins.rs:409-490`):
```rust
pub struct GoSdk { module: String, dir: String }
impl GoSdk {
    pub fn new() -> Self
    pub fn module(mut self, module: impl Into<String>) -> Self   // single source of truth for package name (rule 3)
    pub fn to(mut self, dir: impl Into<String>) -> Self
}
impl Default for GoSdk { fn default() -> Self { Self::new() } }
impl Target for GoSdk {
    fn generate(&self, ir: &ApiGraph, out: &mut Artifacts, _cx: &Cx) -> Result<(), CoreError> {
        // empty module → CoreError::Config; empty dir → CoreError::Config
        let package = sdk_package(&self.module)?;
        let bundle = crate::gosdk::generate(ir, &package, &ir.base_path)?;   // ← PySdk: crate::pysdk::generate
        let dir = self.dir.trim_end_matches('/');
        for (name, contents) in crate::gosdk::split_bundle(&bundle) {        // ← PySdk: crate::pysdk::split_bundle
            // unsafe-name guard: reject is_empty()/'/'/'\\'/".." → CoreError::SdkGen
            out.write(format!("{dir}/{name}"), contents);
        }
        Ok(())
    }
    fn output_anchors(&self) -> Vec<String> {
        if self.dir.is_empty() { Vec::new() } else { vec![self.dir.trim_end_matches('/').to_string()] }
    }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.gnr8/` Pipeline config → PySdk target | The module/dir strings come from user config code; an empty/invalid value must yield a typed `Config` error, not a panic or garbage path. |
| bundle frame name → Artifact path | The frame name is joined onto the output dir; a separator-bearing name could escape it. |
| generated output dir → re-analysis | The generated `*.py` sit in the source tree; without a loop-safety anchor the pipeline would re-ingest them. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | `PySdk::generate` frame-name → Artifact path | mitigate | Reuse the identical GoSdk unsafe-name guard (reject `is_empty()`/`/`/`\`/`..`) → `CoreError::SdkGen` before `out.write`. |
| T-03-02-02 | Elevation of Privilege / loop safety | re-ingesting generated `*.py` | mitigate | `PySdk::output_anchors()` returns the output dir so `lifecycle::exclude_output_paths` drops it from analysis (twin of GoSdk). |
| T-03-02-03 | Spoofing / correctness | second source for the package name | mitigate | Reuse the single `sdk_package` derivation + `ir.base_path` only (rule 3) — no fallback, no re-derivation. |
| T-03-02-04 | Denial of Service / robustness | unconfigured target | mitigate | Empty module/dir → `CoreError::Config` (typed); no `unwrap`/`expect`/`panic` (RUST-04). |
| T-03-02-05 | Information Disclosure / correctness | non-deterministic Artifacts | mitigate | `Artifacts::write` keeps files path-sorted; determinism unit test asserts two runs byte-identical. |
| T-03-02-SC | Tampering | npm/pip/cargo installs | mitigate | None added (no external packages — RESEARCH). Any new dependency is a defect (rule 2). |
</threat_model>
