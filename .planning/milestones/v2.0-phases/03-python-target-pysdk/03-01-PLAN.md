exhaustive type match, required-first dataclass ordering, the four file markers, and determinism — all
WITHOUT any `python3` subprocess (pure string emission).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| neutral IR → emitted Python source | The graph drives `format!` string emission; an un-representable or malformed fact must surface as a typed error, never silent/garbage Python. |
| frame name → on-disk file path (write_to_dir) | A bundle frame name is joined onto a caller dir; a name with a path separator could escape the dir. |
| generated SDK runtime → request URL (emitted code) | A path-param value interpolated into a URL is untrusted at the SDK's runtime. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | `pysdk::write_to_dir` frame-name join | mitigate | Reuse the gosdk name guard verbatim: reject `is_empty()`/`/`/`\`/`..` → `CoreError::SdkGen` before `dir.join`. |
| T-03-01-02 | Tampering | generated `client.py` path interpolation | mitigate | Emit `urllib.parse.quote(str(value), safe="")` around every interpolated path segment (twin of Go `url.PathEscape`, ASVS V5). |
| T-03-01-03 | Tampering (supply chain) | generated SDK imports | mitigate | Emit only stdlib imports (`urllib`/`json`/`dataclasses`/`enum`/`typing`); the Wave-3 hermetic test greps the output for `requests`/`httpx` and asserts absent. No template engine, no Rust crate added (rule 2). |
