## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Rust host → node subprocess | host spawns `node index.js <target_dir>`; target_dir is attacker-influenceable path data |
| node sidecar → target source files | target `*.ts` is untrusted input — parsed as DATA, never executed |
| sidecar stdout → host deserialize | JSON facts crossing the language boundary; host must validate strictly |
| npm registry → tsextract/node_modules | the carve-out's single dependency supply chain |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | run_tsextract subprocess spawn | mitigate | `Command::new("node").args(["index.js", target_dir])` — discrete args, no `sh -c`, no interpolation (mirror run_pyextract T-02-01); acceptance grep asserts no shell |
| T-04-02 | Denial of Service | node toolchain absent | mitigate | typed `TypeScriptToolchainMissing` (no panic, RUST-04); W3 adds the test skip-guard |
| T-04-03 | Tampering | malformed sidecar JSON | mitigate | host deserializes via serde_json into the `deny_unknown_fields` facts DTO -> `FactsParse`, never a partial graph |
| T-04-04 | Malicious Code / Supply Chain | tsextract/node_modules | mitigate | exactly ONE dep (`typescript`, canonical Microsoft compiler, zero transitive deps); pinned exact version + committed lockfile; acceptance asserts deps.length===1; gnr8-core adds zero crates |
| T-04-SC | Tampering | npm install | mitigate | the sole install is `typescript` (VERIFIED 5.9.3, 0 vulnerabilities, no malicious postinstall per RESEARCH Package Legitimacy Audit).
