<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.gnr8/` config (module/dir strings) → TsSdk Target | User-supplied config strings drive file output paths; an unsafe name must not escape the output dir. |
| IR base_path/package derivation → emitted URLs/package | Must come from a single source (rule 3), never a second ad-hoc derivation. |
| Generated SDK output dir → next pipeline run | The output must be a loop-safety anchor so a re-run never re-ingests the generated `.ts`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering/Elevation | `TsSdk::generate` write path | mitigate | Reject any split-bundle frame name containing `/`, `\`, `..`, or empty before `out.write` (verbatim clone of the PySdk Target guard). |
| T-05-02-02 | Spoofing/Tampering | package + base_path derivation | mitigate | `sdk_package(self.module)` + `ir.base_path` are the ONLY sources (rule 3); no second derivation that could drift from the OpenAPI lowering. |
| T-05-02-03 | DoS (loop) | output dir re-ingestion | mitigate | `output_anchors()` returns the output dir so the lifecycle excludes generated `.ts` from the next source scan. |
| T-05-02-04 | DoS | unconfigured/malformed target | mitigate | Empty module/dir return typed `CoreError::Config`; no production unwrap/expect/panic. |
| T-05-02-SC | Tampering | npm/cargo installs | mitigate | This plan installs nothing (no new crate, no npm action; generated SDK dependency-free). Slopcheck N/A per RESEARCH Package Legitimacy Audit. |
</threat_model>
