<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user filesystem → `gnr8 init` | init writes into the user's project root (cwd); must never clobber existing user files |
| `.gnr8/config.toml` → Config parser | user-authored TOML is untrusted text crossing into typed Rust state |
| crates.io → workspace build | three new third-party runtime crates enter the dependency tree |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | `workspace::init` write path | mitigate | `OpenOptions::create_new(true)` write-if-absent; AlreadyExists → skip, never overwrite (D-01 idempotency); no path comes from user input — fixed `.gnr8/` subtree |
| T-04-01-02 | Denial of Service | `config::parse` on malformed/huge TOML | mitigate | `#[serde(deny_unknown_fields)]` + typed `CoreError::Config` (no unwrap/panic, RUST-04); a bad config is a clean error + exit, never a crash |
| T-04-01-03 | Tampering / Elevation | unknown/typo config keys silently ignored | mitigate | `deny_unknown_fields` rejects unrecognized keys so a misspelled `output_dir` fails loudly instead of silently mis-generating |
| T-04-01-SC | Tampering | blake3 1.8 / toml 1.1 / notify-debouncer-full 0.7 installs | accept | RESEARCH Package Legitimacy Audit: all three are long-lived, high-download (30.9M / 159M / 2.9M), source-backed crates from known orgs (BLAKE3-team, toml-rs, notify-rs), slopcheck [OK]; versions pinned to researched stable (no rc); no [ASSUMED]/[SUS]/[SLOP] → no blocking human checkpoint required |
</threat_model>
