The seam to implement (crates/gnr8-core/src/sdk/mod.rs):
  pub fn generate(graph: &crate::graph::ApiGraph) -> Result<String, crate::CoreError>

The ApiGraph the seam consumes — see the same struct layout cited in 03-01's <interfaces>:
  ApiGraph { module, operations: Vec<Operation>, schemas: Vec<Schema>, diagnostics: Vec<Diagnostic> }
  Operation { id, method, path, router_path, handler, summary, tags, secured, security_schemes,
              params: Vec<Param>, request_body: Option<SchemaRef>, responses: Vec<Response>, ... }
  Schema { id, name, kind ("object"|"enum"), fields: Vec<Field>, enum_values, ... }
  Field { json_name, required, optional, schema: SchemaType, ... }
  SchemaType { kind, format ("uuid"|"date-time"|"int64"), items: Option<Box>, ref_id, additional_properties }

The CoreError variants this plan USES (already added by 03-01 in error.rs — do NOT edit error.rs):
  CoreError::SdkGen { message: String }, CoreError::GoFmt { code: Option<i32>, stderr: String }

The discrete-arg subprocess pattern to mirror for gofmt (crates/gnr8-core/src/analyze/helper.rs):
  Command::new("go").args([...]).current_dir(...).output().map_err(|source| GoToolchainMissing { source })?;
  if !output.status.success() { return Err(...) }   // non-zero exit → typed error, never panic

Go SDK type mapping (TARGET-API.md §4 — already encoded in the graph SchemaType facts):
  string→string, bool→bool, int/int64→int64, float64→*float32 (narrowed; diagnostic already in graph),
  uuid.UUID→string, time.Time→time.Time, *T/omitempty→pointer/omitempty, []T→[]T,
  map[string]any→map[string]any, named string+consts→typed string newtype, nested struct→model type.

## Trust Boundaries

| Boundary | Description |
