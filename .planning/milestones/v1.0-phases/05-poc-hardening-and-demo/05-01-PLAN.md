## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user config → doctor | `.gnr8/config.toml` is read by `config::load`; paths in it drive `build_graph`/`plan_only` targets |
| doctor → `go` subprocess | the Go-toolchain probe spawns the external `go` binary |
| bench.sh → filesystem | the script copies the fixture and writes generated outputs into a scratch dir |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering | `probe_go` spawning `go` | mitigate | Pass `go`/`version` as DISCRETE `Command` args, never `sh -c` (reuses helper.rs discipline); a spawn `io::Error` is treated as "absent", never panics. |
| T-05-01-02 | Denial of Service | `run_doctor` on a machine without Go or with a corrupt manifest | mitigate | `build_graph`/`plan_only` `Err` → graceful `None` (no `?`/unwrap); `manifest::load` already returns an empty default on corrupt/absent; doctor REPORTS the condition (Pitfall 4), no crash (RUST-04). |
| T-05-01-03 | Tampering | bench.sh mutating committed state | mitigate | Operate exclusively in a `mktemp -d` scratch copy with `trap 'rm -rf' EXIT`; never `init`/`generate` in `fixtures/goalservice` (Pitfall 2). |
| T-05-01-04 | Information Disclosure / Tampering | `doctor --json` output | accept | serde_json serialization (no manual string building), same as existing `--json` paths; output is local non-secret project health. |
| T-05-01-05 | Tampering | config output paths during a bench generate | mitigate | Already guarded by `lifecycle::safe_output_path` (rejects `..`/absolute/Windows-prefix) — unchanged this phase; the bench runs in an isolated scratch dir regardless. |
| T-05-01-SC | Tampering | npm/pip/cargo installs | accept | No new packages this phase (RESEARCH Package Legitimacy Audit: zero new dependencies — std + already-vendored crates + bash only). No install task exists; nothing to gate. |
