## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Rust analyzer → `go` subprocess | The target dir string crosses into a spawned process. Passed as a discrete `Command` arg, never a shell string. |
| `go/packages` → target Go build system | `go/packages` invokes `go list`/build on the target module for type info. For the PoC the only target is the trusted in-repo `fixtures/goalservice`; arbitrary untrusted modules are a documented post-PoC boundary. |
| goextract stdout → Rust serde | The helper's JSON is parsed by serde with `deny_unknown_fields`; malformed/forward-incompatible JSON is rejected, not trusted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering/Elevation | target-dir arg to `go run` | mitigate | Pass `target_dir` as a discrete `Command::args(["run",".",target])` element; never `sh -c`, never string-interpolate (RESEARCH Security Domain). |
| T-02-02 | Elevation | `go/packages` invoking the target build system | accept | PoC target is the trusted in-repo fixture; `go/packages` type-checks (does not execute) target code. Documented limitation for untrusted modules (post-PoC). |
| T-02-03 | DoS | panic on malformed helper output | mitigate | Typed `CoreError::{GoToolchainMissing,HelperExit,FactsParse}` + `?`; clippy `unwrap_used=deny` enforces no-panic (GO-06). |
| T-02-04 | Tampering | supply-chain (helper dep) | mitigate | Only dep is official `golang.org/x/tools` (CITED, Go team); commit `goextract/go.sum` for reproducible CI build. |
| T-02-05 | Tampering | malformed/forward-incompatible JSON across the boundary | mitigate | `#[serde(deny_unknown_fields)]` on every facts DTO (Security V5). |
| T-02-SC | Tampering | go module install (`go get golang.org/x/tools`) | mitigate | Package is `[CITED]` (RESEARCH Package Legitimacy Audit: official Go team module, confirmed via `go list -m -versions` + pkg.go.dev).
