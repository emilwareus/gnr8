# Plan 05-02: Common Request Media Types

## Goal

Make generated Go, Python, and TypeScript SDKs encode the common request media types that block
production adoption without adding server stubs or template overrides.

## Scope

- Classify request body media types once in the shared SDK emitter layer.
- Preserve JSON behavior while adding explicit SDK encoders for:
  - `text/plain`
  - `application/x-www-form-urlencoded`
  - `multipart/form-data`
  - `application/octet-stream`
- Keep binary downloads on the existing response path.
- Emit clear SDK generation diagnostics for unsupported media types or incompatible schema shapes.

## Acceptance

- `MEDIA-01`: OpenAPI and SDKs support JSON, text, form-urlencoded, multipart, and binary upload/download.
- Unsupported request media types fail during generation instead of silently JSON-encoding.
