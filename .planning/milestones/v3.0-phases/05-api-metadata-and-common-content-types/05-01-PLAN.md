# Plan 05-01: Operation Metadata, Examples, And Error Documentation

## Goal

Add selector-based operation documentation transforms that feed both OpenAPI 3.1 and generated SDK
docs, without treating documentation metadata as source-derived route facts.

## Scope

- Add graph-owned `operation_docs` policies keyed by operation id.
- Add a `DocumentOperation` transform with selector matching for summaries, descriptions, tags,
  deprecation, response descriptions, named media examples, and documented JSON error responses.
- Lower operation metadata/examples into OpenAPI YAML and JSON.
- Surface the same metadata in generated SDK reference and OpenAPI Generator-compatible markdown docs.

## Acceptance

- `META-01`: selector-based transforms configure summary, description, deprecation, tags, response
  descriptions, and documented error responses.
- `META-02`: named request/response examples attach to operation media types.
- `META-03`: OpenAPI and SDK docs propagate metadata, examples, deprecation notes, tags, and documented
  error responses.
