# Phase 4 Plan 2 — SDK Runtime Timeouts Retries And Hooks

Status: Complete

## Intent

Render production SDK runtime ergonomics from the shared metadata added in Plan 04-01: client and
per-request timeout/retry controls, conservative retry behavior, idempotency-key preservation, and
transport hooks with operation context.

## Scope

- Add optional/variadic request options to generated Go, Python, and TypeScript operation methods while
  preserving raw operation methods.
- Add client-level timeout and max-retry configuration plus per-request overrides.
- Retry only network errors and retryable statuses (`408`, `429`, and `5xx`), gated by safe methods,
  explicit idempotency metadata, or an explicit unsafe-retry policy.
- Preserve consumer-supplied idempotency keys across retries for explicitly idempotent operations.
- Add request, response, and error hooks carrying operation id, method, path template, resolved URL,
  headers, request metadata, status, and response headers.

## Non-Goals

- Pagination helpers; that is Plan 04-03.
- OAuth/token-refresh/mTLS/signing flows.
- Compatibility-profile parity for generated OpenAPI Generator request builders.
