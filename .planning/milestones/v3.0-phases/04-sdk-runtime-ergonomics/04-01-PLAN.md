# Phase 4 Plan 1 — Runtime And Pagination Metadata

Status: Complete

## Intent

Add explicit code-as-config metadata for SDK runtime policy, operation idempotency, and pagination
without inferring behavior from parameter names or source comments.

## Scope

- Add graph-owned defaults for SDK runtime policy, operation runtime metadata, and pagination metadata.
- Add transforms that configure client defaults, operation idempotency, cursor pagination, and page/offset
  pagination by selector.
- Lower those facts into the shared SDK model so every target reads the same operation policy.
- Add focused tests for deterministic defaults, transform matching, and model propagation.

## Non-Goals

- Generated retry/timeout/hook runtime code; that is Plan 04-02.
- Generated pagination helpers; that is Plan 04-03.
- Server stubs or inference from legacy/vendor OpenAPI extensions.
