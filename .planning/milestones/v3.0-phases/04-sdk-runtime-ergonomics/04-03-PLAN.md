# Phase 4 Plan 3 — SDK Pagination Helpers

Status: Complete

## Intent

Render pagination helpers from explicit `PaginationPolicy` metadata without inferring API-specific
pagination behavior from route names or response shapes.

## Scope

- Emit helpers only for operations with configured pagination policies.
- Support cursor, page-number, and offset/limit policies from the shared graph metadata.
- Preserve the raw generated operation method for every paginated operation.
- Provide page helpers and item helpers where the response item field is statically discoverable.
- Verify generated Go and Python helpers at runtime, and TypeScript helper output with emitter tests.

## Non-Goals

- Broad inference of pagination fields from OpenAPI schemas.
- Async/concurrent page prefetch.
- Compatibility-profile pagination builders.
