# Phase 3 Plan 1 — SDK Readiness And Package Metadata

Status: Complete

## Intent

Make generated SDK targets adoption-ready by exposing `doctor --json` readiness entries and ensuring
package metadata exists for the broad SDK package managers without registry publishing automation.

## Scope

- Add a stable `sdk_readiness` section to `gnr8 doctor --json` and human output.
- Infer configured SDK/OpenAPI targets from the emitted artifact bundle and validate generated files
  in an isolated temporary materialization.
- Keep source diagnostics informational unless SDK/OpenAPI readiness itself fails.
- Emit Python `pyproject.toml` package metadata and strengthen TypeScript `package.json`.
- Cover the readiness report shape and Python package metadata with focused tests.

## Non-Goals

- Registry uploads, credentials, or full package publishing automation.
- Exhaustive OpenAPI semantic validation beyond parseability and obvious local `$ref` checks.
- Older generator compatibility profiles beyond the existing `compat` surfaces.

## Verification

- `cargo test -p gnr8-cli doctor -- --nocapture`
- `cargo test -p gnr8 pysdk -- --nocapture`
- `cargo test -p gnr8 validate_openapi_artifact_checks -- --nocapture`
- `cargo test -p gnr8 generated_sdk -- --nocapture`
- `cargo test -p gnr8 sdk_surface -- --nocapture`
- `cargo clippy -p gnr8-cli --all-targets -- -D warnings`
- `cargo clippy -p gnr8 --all-targets -- -D warnings`
