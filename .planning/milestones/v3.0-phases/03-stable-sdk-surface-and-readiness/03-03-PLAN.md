# Phase 3 Plan 3 — Stable Surface Grouping Selectors

Status: Complete

## Intent

Close the stable SDK surface requirement by making operation grouping cover the broad adoption
selectors: imported/source tags, path prefixes, source module/package prefixes, and explicit operation
IDs.

## Scope

- Add `GroupOperations` rules for existing source/imported groups and source path/module prefixes.
- Keep first-match-wins deterministic behavior.
- Add focused tests showing source tags are preserved/remappable and source path prefixes can assign
  SDK/OpenAPI groups.

## Non-Goals

- New source annotations or server-stub semantics.
- Compatibility with legacy OpenAPI specs beyond the existing importer behavior.
