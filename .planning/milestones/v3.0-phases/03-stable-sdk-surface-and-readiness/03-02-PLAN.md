# Phase 3 Plan 2 — Package Metadata Configuration And Publishing Recipes

Status: Complete

## Intent

Expose package metadata configuration across SDK targets and generate local publishing recipes so SDKs
can be prepared for registries without `gnr8` handling uploads or credentials.

## Scope

- Add a shared package metadata builder for registry/distribution name, version, description, license,
  repository, homepage/docs URLs, and keywords.
- Thread package metadata into TypeScript `package.json` and Python `pyproject.toml`.
- Emit `PUBLISHING.md` recipes for Go, Python, and TypeScript package outputs.
- Keep `source_only()` as the escape hatch for source files without docs/package artifacts.

## Non-Goals

- Registry publishing automation.
- Credential storage or CI secret generation.
- Full package-manager linting beyond the readiness checks already added in Plan 03-01.

## Verification

- `cargo test -p gnr8 package_metadata -- --nocapture`
- `cargo test -p gnr8 pysdk_target_emits_pyproject_metadata -- --nocapture`
- `cargo test -p gnr8 tssdk_fetch_compat_emits_package_metadata_and_source_only_can_disable_it -- --nocapture`
- `cargo test -p gnr8 generated_sdk -- --nocapture`
- `cargo clippy -p gnr8 --all-targets -- -D warnings`
