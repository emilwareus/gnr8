# Phase 2 Plan 2: Bearer And Basic Auth

## Goal

Support HTTP bearer and basic authentication as first-class security schemes across `.gnr8` transforms,
OpenAPI 3.1 output, imported OpenAPI sources, and generated Go/Python/TypeScript SDKs.

## Requirements Covered

- AUTH-01: Global/per-operation auth lowers consistently into OpenAPI and SDKs.
- AUTH-02: SDK consumers can configure bearer token or basic auth once at client construction.
- AUTH-03: Operation-level public overrides do not send inherited credentials.
- AUTH-04: Runtime/unit tests verify outgoing auth behavior.
- AUTH-05: Unsupported schemes still produce clear diagnostics.

## Implementation

1. Add `ApplySecurity::bearer(id)` and `ApplySecurity::basic(id)`.
2. Lower graph schemes with `kind = "http"` and `name = "bearer" | "basic"` into OpenAPI
   `type: http` plus `scheme: bearer|basic`.
3. Import OpenAPI `http` bearer/basic schemes into the graph.
4. Extend shared SDK auth resolution to include operation-scoped bearer/basic requirements.
5. Emit Go/Python/TypeScript client constructor options and per-operation Authorization behavior.
6. Add focused lowering, resolver, emitter, and Go runtime smoke tests.

## Verification

- `cargo fmt --all --check`
- `cargo test -p gnr8 sdk::emit_common`
- `cargo test -p gnr8 lower::`
- `cargo test -p gnr8 openapi_source`
- `cargo test -p gnr8 --test sdk_compile --test pysdk_compile --test tssdk_compile`
