# Phase 2 Plan 4: Cross-Target Auth Runtime Smoke

## Goal

Close the Phase 2 verification gap by proving generated SDK auth wiring at runtime for every target
whose local toolchain is available in this workspace.

## Requirements Covered

- AUTH-04: Runtime smoke tests verify outgoing auth headers/query params across generated SDKs where
  local toolchains are available.

## Implementation

1. Keep the existing Go runtime smoke for query API-key, bearer, and basic auth.
2. Add a Python generated-SDK runtime smoke over a synthetic auth graph using stdlib `http.server`.
3. Exercise query API-key, bearer token, and basic auth through the generated Python `Client`.
4. Leave TypeScript runtime execution to the existing skip-if-toolchain-present test harness; this
   workspace currently lacks Node/tsc, so TypeScript remains covered by generator assertions and compile
   gates when the toolchain exists.

## Verification

- `cargo test -p gnr8 generated_sdk -- --nocapture`
- `cargo test -p gnr8 auth -- --nocapture`
- `cargo clippy -p gnr8 --all-targets -- -D warnings`
