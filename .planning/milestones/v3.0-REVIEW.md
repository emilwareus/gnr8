# v3.0 Code Review

Scope: `origin/main...HEAD`
Depth: review/fix loop
Date: 2026-07-09

## Result

No remaining findings in the reviewed v3.0 SDK adoption scope after the fix pass.

## Resolved Findings

### HIGH - TypeScript retried hook failures as if they were network failures

Status: Resolved

Generated TypeScript previously wrapped request hooks, `fetch`, response hooks, and non-2xx error-hook
dispatch inside one retry catch. That made request-hook validation errors and response-hook failures
retryable for idempotent/retryable operations.

Fix: `_request` now runs request hooks before the fetch retry catch, catches only `fetchFn` failures for
transport retries, and runs response hooks after the fetch retry path. Hook failures still dispatch error
hooks once, then rethrow without another fetch attempt.

Validation: the TypeScript runtime emitter test now asserts the request hook block is before the fetch
retry catch and that the hook-failure rethrow path exists before fetch retry handling.

### MEDIUM - Go form-url-encoded arrays diverged from Python and TypeScript

Status: Resolved

Generated Go form encoding previously wrote every field through `values.Set(name, formValue(...))`, and
`formValue` joined slices with commas. Python and TypeScript already encoded array-valued form fields as
repeated keys.

Fix: Go form encoding now routes fields through `addFormField`, which uses `values.Add` for slices and
arrays while preserving scalar `Set` behavior.

Validation: the Go media runtime smoke test now sends `tags: []string{"sdk", "media"}` and asserts the
server receives repeated `tags` values.

### MEDIUM - Multipart byte arrays were typed as supported but not encoded as repeated file parts

Status: Resolved

TypeScript exposed multipart `array<bytes>` fields as supported, but arrays fell through to text encoding.
Go `[][]byte` and Python lists of `bytes` had the same effective failure mode.

Fix: TypeScript, Go, and Python multipart encoders now expand array/list/tuple values into repeated parts.
Byte values are emitted as file parts, preserving scalar text-field behavior for non-byte values.

Validation: Go, Python, and TypeScript media tests now include multipart `files` with two byte values. Go
and Python runtime tests assert that two repeated file parts arrive and contain `part-one` and `part-two`;
the TypeScript consumer typecheck also exercises the generated array-of-bytes surface.
