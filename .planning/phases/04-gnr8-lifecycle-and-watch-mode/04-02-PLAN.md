<objective>
Build the ownership manifest and the pure write-decision core (WS-04, WATCH-01), apply naming overrides
(WS-03), and wire `gnr8 generate` + `gnr8 check` to the real deterministic pipeline. This delivers the two
headline falsifiable guarantees: **no silent clobbering** (a hand-edited generated file is protected unless
`--force`) and **no-op = no write** (byte-identical regeneration touches zero files).

Purpose: the pure `plan_writes(new_outputs, manifest, on_disk) -> WritePlan` function is the heart of the
phase — every `generate`, `check`, and (in 04-03) watch-triggered regeneration routes through it. Keeping it
pure (no I/O; on-disk bytes injected via a closure) makes the full truth table exhaustively unit-testable
without touching the filesystem (RESEARCH Pattern 2, Pitfall 3).

Output: `manifest` + `lifecycle` modules, naming-override application, a `regenerate(config, force)` entry
point returning a `GenerateOutcome` with counts, the wired `generate`/`check` CLI arms, and the full
plan_writes truth-table tests appended to `tests/lifecycle.rs`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user config output paths → `apply_writes` | `output.openapi` / `output.sdk_dir` are user-controlled strings that become filesystem write targets |
| on-disk generated files → `plan_writes` | files at output paths may have been hand-edited or pre-existed; must not be clobbered silently |
| `manifest.json` (cache) → ownership decisions | a corrupt/tampered manifest must degrade safely, never crash or mask a destructive write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering / Elevation | `apply_writes` output path from config (`sdk_dir = "../../etc"`) | mitigate | resolve each output path against project_root and REJECT escapes (`..`, canonicalize outside root) → `CoreError::Io`; extends the `sdk::write_to_dir` frame-name path-safety (T-03-03) to user-supplied paths (Security V5) |
| T-04-02-02 | Tampering | silent clobber of a hand-edited generated file | mitigate | `plan_writes` classifies on-disk-hash ≠ recorded-hash as `UserEdited` → warn (stderr) + skip unless explicit `--force`; pre-existing un-tracked output (present on disk, absent from manifest) also protected (Pitfall 5) — the headline WS-04 guarantee |
| T-04-02-03 | Denial of Service | corrupt/huge `manifest.json` crashes generate | mitigate | corrupt manifest loads as empty default (regenerate-from-scratch), never panics; only a genuine read I/O error is a typed `CoreError::Manifest` (RUST-04, no unwrap) |
| T-04-02-04 | Repudiation | user cannot tell which files were protected | mitigate | every `skipped` (UserEdited) file is named in a stderr warning line so the protection is visible, not silent |
| T-04-02-SC | Tampering | blake3 used as content fingerprint | accept | blake3 is a non-secret content fingerprint (integrity-by-comparison), NOT a security primitive; pinned, slopcheck [OK] in 04-01; `check` re-derives from disk so a tampered manifest at worst forces a regeneration |
</threat_model>
