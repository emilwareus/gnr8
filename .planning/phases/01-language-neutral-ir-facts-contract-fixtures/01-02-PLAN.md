<objective>
Make the two IR consumers — OpenAPI lowering (`lower/`) and the Go SDK target (`gosdk/`) — consume the
neutral `Type` enum from Plan 01 with an exhaustive `match` (no `_ =>` catch-all, no per-language
branch), keeping all Go-specific type mapping inside the `gosdk` target. Fix the two known semantic
bugs flagged in RESEARCH: (a) optional-vs-nullable conflation in `gosdk/emit.rs maybe_pointer`; (b) the
factually-wrong OpenAPI-3.1 comment in `lower/model.rs:138` — 3.1 DOES use `type:["T","null"]` — and
render nullability via that array form. Then deliberately re-accept the existing `goalservice`
snapshots so the Go pipeline stays GREEN.

Purpose: IR-03 — the Rust lowering → OpenAPI → SDK pipeline must consume the neutral IR unchanged across
all languages with zero per-language branches. The closed enum from Plan 01 currently breaks these two
consumers' compilation on purpose; this plan resolves those compile errors the right way (exhaustive
arms), not with shims.

Output: `lower/` and `gosdk/` compile and pass against the neutral enum; nullable and optional are
distinct end-to-end; the Go-fixture snapshots are green with reviewed, intended churn.
</objective>
