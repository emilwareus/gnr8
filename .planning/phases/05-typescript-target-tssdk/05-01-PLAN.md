<objective>
Create the `crates/gnr8-core/src/tssdk/` module — the IR→TypeScript SDK emitter — as a pure
rename-and-clone of `crates/gnr8-core/src/pysdk/`. It produces a deterministic, dependency-free
4-file TypeScript SDK bundle from the neutral `ApiGraph` IR: typed `interface` models,
string-literal-union enums (`type X = "a" | "b"`), a `fetch`-based `Client` with an injectable
transport, and a typed `ApiError extends Error`.

Purpose: TSSDK-01 — establish the fourth language path's SDK target as a pure IR→string twin, with
zero new gnr8-core crates and zero runtime deps in the generated SDK (CLAUDE.md rule 2).
Output: `tssdk/{bundle,emit,mod}.rs` + `pub mod tssdk;` in `lib.rs`, all unit-tested.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| IR (ApiGraph) → emitted TS strings | The IR is program-controlled, but a malformed/dangling fact must produce a typed error, never a panic or a silently-wrong type. |
| bundle frame name → on-disk path (write_to_dir) | Frame names are program-generated, but a malformed name must not traverse out of the target dir. |
| emitted SDK source → consumer runtime | The generated TS must carry zero runtime dependencies (supply-chain). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering/Elevation | `tssdk::write_to_dir` frame name | mitigate | Reject any name containing `/`, `\`, `..`, or empty before `dir.join` (verbatim clone of pysdk's guard). |
| T-05-01-02 | Tampering | emitted TS request construction | mitigate | Path params interpolated via `encodeURIComponent(String(x))` (V5 URL-injection mitigation, the `urllib.quote(safe='')` analog); query via `URLSearchParams`. |
| T-05-01-03 | Tampering (supply chain) | emitted SDK imports | mitigate | Emitters use only the `fetch` global + `JSON`/`URLSearchParams`/`encodeURIComponent` built-ins — no `axios`/`node-fetch`/`@types` import emitted (grep-gated in Plan 03). |
| T-05-01-04 | DoS | `ts_type` / emitters on malformed IR | mitigate | Dangling `$ref`, inline object, path-token/param mismatch, and duplicate schema name all return typed `CoreError::SdkGen`; `fmt` errors folded via `sink`; no production unwrap/expect/panic. |
| T-05-01-SC | Tampering | npm/cargo installs | mitigate | This plan installs nothing (Rust stdlib only; generated SDK dependency-free; `typescript` already vendored). No package-manager action — slopcheck N/A per RESEARCH Package Legitimacy Audit. |
</threat_model>
